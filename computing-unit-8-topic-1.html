Assemblers

An assembler converts low level assembly language (e.g. Little Man Computer) into machine code.



Interpreters

An interpreter converts high-level code one line at a time, into machine code and executes it. It is easier to debug interpreted code but it is slower to execute compiled code.



Compiler

A compiler converts high level code into machine code for execution at a later time (the entire program is converted, not one line at a time as with an interpreter). It is slower to compile code than to interpret it but it can be executed faster once it has been compiled.





Stages of Compilation


1 - Lexical Analysis

The term 'lexical' refers to words and phrases. High level language needs to be broken down into tokens for the next stage of analysis.

  • Unnecessary spaces or comments are removed

  • Keywords, constants and identifiers are replaced by 'tokens'

  • A symbol table is created, holding the addresses of variable, labels and subroutines

    • 2 - Syntax Analysis

    The term 'syntax' refers to sentence structure.

  • Tokens are checked to see if they match the spelling and grammar expected of the programming language. This is done by parsing each token (checking that it uses the correct syntax rules)

  • If a syntax error is discovered, compilation stops and an error message is displayed

    • 3 - Semantic Analysis

    The term 'semantic' refers to logic. Variables are checked in this stage to ensure they are used correctly.

  • Variables are checked to make sure they are correctly declared and use a valid data type. (e.g. strings are not assigned an integer value)

  • Operations are checked to make sure they are legal for the variable type being used (e.g. not attempting to store the result of division as an integer)

    • 4 - Code Generation

  • Machine code is generated

  • This code may be optimised so it is more efficient or uses less of the computer's resources
