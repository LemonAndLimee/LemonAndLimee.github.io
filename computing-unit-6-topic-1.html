High Level Languages

A high level language is a programming language that allows code to be written. It is similar to a natural human language, such as English.

Some programmers prefer to use high level programming languages, as they are easier to understand, learn and program. Their commands are similar to natural languages like English and identifiers can be long and meaningful. High level programming languages also allow the use of powerful commands that perform quite complex tasks.

However, high level languages cannot be directly performed by the CPU, and must be translated to a machine code first.

Low Level Languages

Assembly language is a low level language consisting of abbreviations and symbols. It is more simple than high level languages, but easier to understand than pure machine code. It is converted to machine code using an assembler.

Programming in a low level language, such as assembly language, requires knowledge of the internal structure of the CPU and is therefore very specialised.

Source code produced in a low level language is not portable, but it can be very efficient and the programs can be made to run faster than programs produced using a high level language.








Machine Code

Machine language is made up entirely of binary or hexadecimal instructions. Examples of machine code instructions are opcodes and operands.

It is rarely typed directly into the computer as it is very long, and takes time, and is susceptible to typing errors. It is most often translated from high level languages.
