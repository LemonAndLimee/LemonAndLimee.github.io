Translators
Assembler


An assembler translates low level languages into machine code.

It is the most efficient translator, as the language is at a lower level.

However, it requires knowledge of assembly language, which can be difficult for humans to understand.





Interpreter


An interpreter converts high level code into machine code, one line at a time.

This means it has a slower execution speed, but is easier to debug as errors can be identified easier.





Compiler


A compiler translates high level code into machine code. It converts the entire code at once.

This means it has a quicker execution speed.

However, it is harder to debug, and has to be re-compiled every time the code changes.





Stages of Compilation

There are four main stages of compilation using a compiler.

1. Lexical Analysis

Unnecessary spaces and comments are removed Keyword, constants and identifiers are replaced with 'tokens' A symbol table is created, storing the addresses of all variables, labels and subroutines used.


2. Syntax Analysis

The tokens are checked against the programming language's spelling and grammar rules. If a syntax error is found, an error message is displayed and compilation stops.


3. Semantic Analysis

Variables are checked that they are correctly declared and use a valid data type. Operation checks ensure they are legal for the variable type being used.


4. Code Generation

Machine code is generated. The code may be optimised so it is more efficient or uses less of the computer's resources.
