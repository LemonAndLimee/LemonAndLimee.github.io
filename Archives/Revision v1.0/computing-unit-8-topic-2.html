Programming Errors
Revision
Quiz

Syntax Error

A syntax error is an error in the rules or grammar of the program that prevents the program from being compiled.
E.g. 'Primt' in Python should be 'print'



Logical Error

A logical error is a mistake in the program instructing it to do the wrong thing so that it calculates the wrong output.
E.g. A '+' in a maths sum instead of a '-'.



Rounding Error

A rounding error is where the computer rounds a value to a fixed number of decimal places, resulting in the loss of some information and an inaccurate result.
E.g. 3.225 rounding to 3.23



Runtime Error

A runtime (or execution) error is where the computer unexpectedly stops as the result of an operation during execution.
E.g. Attempting to read line 12 of a program that only has 11 lines.



Linking Error

A linking error is where the compiler cannot find a subroutine. The programmer may have declared it incorrectly or hasn't imported it into the code.
E.g. [With no import] sleep(2)



Truncation Error

A truncation error is where the computer truncates a value to a certain number of decimal points, causing potentially inaccurate calculations.
E.g. 3.125 truncating to 3.12
