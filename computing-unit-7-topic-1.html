Integrated Development Environment
Revision
Quiz

An Integrated Development Environment (IDE) allows the user to write computer programs, using specialised tools to help them program. Examples of these tools are editors, debuggers and linkers.




Editor

An editor allows a programmer to enter, format and edit computer code. Different commands and functions may be different colours to allow a programmer to easily distinguish the different parts of the code. The editor is extremely important because without it, the user would not be able to enter the source code.


Compiler

A compiler converts the high level code into executable machine code for a later time (the entire program is converted, not one line at a time). It is slower to compile code than to interpret it but it can be executed faster once it has been compiled.


Interpreter

An interpreter converts the high level code one line at a time, into machine code and then executes it. It is easier to debug interpreted code as you can run each line at a time, searching for bugs. However, it is slower to execute than compiled code.


Linker

A linker is a program which allows previously compiled code, from software libraries, to be linked together.


Loader

A loader is a program which loads previously compiled code into memory.


Debugger

A debugger helps to locate, identify and rectify errors in a program.


Trace

A trace is a facility which displays the order in which the lines of a program are executed, and possibly the values of variables as the program is being run.


Breakpoint

A breakpoint is a facility which interrupts a program on a specific line of code, allowing the programmer to compare the values of variables against expected values. The program code can then usually be executed one line at a time. This is called single-stepping.


Variable Watch

A variable watch is a facility that displays the current value of any variable. The value can be 'watched' as the program code is single-stepped to see the effects of the code on the variable. Alternatively a variable watch may be set, which will interrupt the program flow if the watched variable reaches a specified value


Memory Inspector

A memory inspector is a facility which will display the contents of a section of memory.


Error Diagnostics

Used when a program fails to compile or to run. Error messages are displayed to help the programmer diagnose what has gone wrong.
